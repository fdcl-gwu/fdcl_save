Basic Introduction

This class is used for easily saving data into a file on a hard disk with C++. Below is an example on how to use this class.

1 #include "fdcl/save.hpp"
2 
3 
4 int main(void)
5 {
6  // initialize the class first
7  fdcl::save log;
8 
9 
10  // define few variables that needs to be written
11  int i = 0;
12  double d = M_PI;
13 
14  Eigen::Matrix<double, 3, 1> J;
15  J.setIdentity();
16 
17 
18  // open the file
19  log.open("test_fdcl_save.txt");
20 
21 
22  // write data
23  log.write(d);
24  log.write(i);
25 
26  // write a new line character
27  log.endl();
28 
29  // write more data
30  log.write(d);
31  log.write(J);
32 
33  // write another new line character to the file
34  log.endl();
35 
36 
37  // save data to the hard drive
38  // skipping this step may not write any data
39  log.close();
40 }

Using the Class

To add this to class to your code, using CMake and cloning it as a git submodule is recommended. This instructions assumes that you are going to add your submodules to a directory named libraries in the main project directory. If your submodule directory is different, make sure to change the path wherever it says libraries. First, add this as a submodule in git.

1 cd 'main/project/dir'
2 git submodule add https://github.com/fdcl-gwu/fdcl_save.git ./libraries/fdcl_save
3 git submodule update --init --recursive

NOTE: Whenever you clone your main project, you must recursively update the submodules:

1 git submodule update --init --recursive

Now, in the main project's CMake file (CMakeLists.txt), do the followings:

1 include_directories(${PROJECT_SOURCE_DIR}/libraries/fdcl_save/include)
2 add_subdirectory(${PROJECT_SOURCE_DIR}/libraries/fdcl_save fdcl_save)

Also, whenever you make a file that uses fdcl_save class, add fdcl_save to the linker:

1 target_link_libraries(name/of/the/library/or/exec
2  PRIVATE fdcl_save
3 )

Then, you can simply call #include "fdcl/save.hpp" in your source/header files in the main directory.

Contributing

  • Anyone is welcome to contribute, but make sure you follow the existing coding style.
  • Make sure to document all your changes/additions with Doxygen style comments.

Generating the Documentation

Document generation is done with Doxygen If you do not have Doxygen, install it first

1 sudo apt-get install -y doxygen graphviz

Use Doxygen to generate the documentation

1 cd docs/Doxygen
2 doxygen Doxygen

This will generate the documentation. Commit and push to update the online documentation. back to contents
